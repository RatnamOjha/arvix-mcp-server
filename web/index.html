Architecture
How it works
New in v2
Vectorless RAG

Traditional RAG embeds every chunk with a neural model and retrieves by cosine similarity. Vectorless RAG replaces the embedding step with BM25 keyword retrieval — the same algorithm that powers Elasticsearch and academic search engines. It excels at exact technical terms like LoRA, RLHF, KV-cache that dense vectors often miss. Multi-query results are fused with Reciprocal Rank Fusion (RRF), then each retrieved chunk is compressed to only the relevant sentences by a sentence-level BM25 pass.

step 01
Query expansion
LLM rewrites query into 3 keyword variants to maximize BM25 recall
step 02
Multi-query BM25
Okapi BM25 scores all chunks for each variant independently
step 03
RRF fusion
Reciprocal Rank Fusion combines 3 ranked lists into one without score normalization
step 04
Contextual compression
Sentence-level BM25 extracts only the 2–3 sentences from each chunk that answer the question
01 —
Claude calls a tool

Claude Desktop connects over stdio via the MCP protocol. Research questions are routed to the appropriate tool as structured JSON.

02 —
ArXiv API + PDF extraction

Live search hits the ArXiv REST API. Fetch downloads PDFs via httpx and uses pdfplumber to extract clean text, capped at 40 pages.

03 —
Chunking → BM25 index

Text is split into 500-word chunks with 50-word overlap. Each chunk is added to the Okapi BM25 index immediately — no encode() call, no model load.

04 —
Multi-query retrieval + RRF

Three query variants are BM25-searched independently. RRF score = Σ 1/(k + rank). The top-2k fused results go to the compression step.

05 —
Cited context returned to Claude

Compressed chunks are formatted as [Source N] citations and returned as structured JSON. Claude generates a grounded, cited answer.

Quick start
Running in four steps
1
Clone the repo
git clone https://github.com/RatnamOjha/arvix-mcp-server cd arxiv-mcp-server
2
Create a virtualenv and install
python -m venv .venv && source .venv/bin/activate pip install -e .
No GPU required. No model download. Dependencies install in under 30 seconds.
3
Add to Claude Desktop config
{ "mcpServers": { "arxiv-research": { "command": "/absolute/path/to/.venv/bin/python", "args": ["-m", "src.server"], "cwd": "/absolute/path/to/arxiv-mcp-server" } } }
Config at ~/Library/Application Support/Claude/claude_desktop_config.json on macOS. Use the full .venv/bin/python path — avoids Anaconda conflicts.
4
Restart Claude Desktop and start researching
"Search for papers on speculative decoding" "Fetch paper 2305.10601 and index it" "What does my library say about KV cache optimization?"

Use the full pipeline
from your terminal
Works on free plan
test_local.py

Claude Pro is only needed for the Claude Desktop chat interface. The entire ML pipeline — ArXiv search, PDF download, BM25 indexing, RAG queries, reading list — runs directly from your terminal with no subscription required. Use test_local.py included in the repo.

→
Search ArXiv live
python test_local.py --search
Queries the live ArXiv API and returns ranked paper results with titles, authors, and IDs.
→
Fetch and index a paper
python test_local.py --fetch --arxiv-id 2005.11401
Downloads the PDF, extracts text, chunks it, builds the BM25 index. No embeddings. ~15 seconds.
→
Query your library with RAG
python test_local.py --query --question "what retrieval method does RAG use?"
Full vectorless RAG pipeline: query expansion → BM25 → RRF fusion → contextual compression → cited answer.
→
Run everything at once
python test_local.py
Runs search, fetch, query, and library listing in sequence. Great for a first end-to-end demo.
